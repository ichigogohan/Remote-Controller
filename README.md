# Remote Controller

スマートフォンのブラウザから、PCの画面確認やマウス・キーボード操作を低遅延かつ安全に行える、Webベースのリモートデスクトップ＆コントローラーシステムです。
専用アプリのインストールは不要で、起動時にターミナルに表示されるQRコードをスマホで読み取るだけで瞬時に接続できます。

---

## 主な機能

- **QRコードによる接続:** ターミナルに表示されたQRコードをスマホでスキャンするだけで自動接続します。
- **2つの機能:**
  - **Trackpad & Key モード:** スマートフォンをPCのトラックパッドおよび、モバイルに最適化されたカスタムキーボードとして利用できます。
  - **Screen Viewer モード:** PCの画面をリアルタイムにミラーリング配信します。
- **低遅延・高パフォーマンス配信:** 画面キャプチャに高速な`mss`ライブラリを採用し、非同期スレッドとOpenCVを組み合わせることで、操作のタイムラグを最小限に抑えています。
- **リアルタイム・テレメトリHUD:** 映像配信画面には、現在の配信「FPS」「Ping (応答速度)」「ネットワーク帯域（Mbps）」がリアルタイムに数値で可視化されます。
- **セキュリティトークン認証:** 起動ごとに安全なランダムURLトークン（`ACCESS_TOKEN`）を自動生成。トークンが一致しない第3者からの不正アクセスを自動的に遮断します。

## 技術スタック

### バックエンド (PCサーバー側)
- **Python 3.12.10**
- **Flask & Flask-SocketIO:** 低遅延な双方向通信（操作イベントの受信）およびWebサーバーの構築。
- **PyAutoGUI:** OSレイヤーでのマウス移動、クリック、キー入力のシミュレーション。
- **OpenCV & mss:** 高速なデスクトップキャプチャと画像圧縮（MJPEGストリーミング）。
- **qrcode:** ターミナル上へのQRコード描画。

### フロントエンド (スマホクライアント側)
- **HTML5 / CSS3 / JavaScript (ES6+)**
- **Tailwind CSS:** モバイルに最適化された、直感的でモダンなダークモードUI。
- **Socket.IO Client:** タッチ操作やキー入力をサーバーへ送信。

## 環境構築と起動方法
### パターンA：実行ファイル (.exe) から起動する場合（推奨）
[リリースノート](https://github.com/ichigogohan/Remote-Controller/releases/)から、最新の RemoteController.exe をインストールし、ダブルクリックして実行します。

### パターンB：Pythonソースコードから起動する場合
### 1. 前提条件
PCに Python 3.x がインストールされていることを確認してください。

### 2.仮想環境の作成
仮想環境（`venv`）を作成し、有効化します。

### 3. 依存関係のインストール
プロジェクトのルートディレクトリで以下のコマンドを実行し、必要なライブラリを一括インストールします。

```Bash
pip install -r requirements.txt
```
### 4. サーバーの起動
以下のコマンドでシステムを起動します。

```Bash
python app.py
```
### 使い方
起動すると、ターミナルが自動でクリアされ、以下のようにネットワーク情報と QRコード が大きく表示されます。

```Plaintext
[*] Server IP: 192.168.1.XX
[*] Control Port: 5001
[*] Stream  Port: 5002

[ QRコードをスキャンして接続してください ]
（ここにQRコードが表示されます）
```

操作したいスマートフォンやタブレットを、PCと同じローカルネットワーク（同じWi-Fi）に接続します。

スマホのカメラ等でターミナルのQRコードをスキャンします。

ポータル画面が開くので、用途に合わせて「Trackpad & Key」または「Screen Viewer」を選択して操作を開始してください。

## システムの特徴（設計について）
本システムは、操作の追従性を最優先するためダブルポート設計を採用しています。

ポート 5001 (操作系): ページのレンダリングおよび、Socket.IOによるマウスやキーイベントの送受信を担当します。

ポート 5002 (映像系): MJPEG形式の映像ストリーミング、およびFPS等のデータを返すテレメトリAPI専用のポートです。

高負荷な映像処理を別ポート（別スレッド）に完全に切り離すことで、画面が動いている最中でもスマホからのタップやスクロール操作がカクつくことなく、滑らかにPCへ反映される構造になっています。
